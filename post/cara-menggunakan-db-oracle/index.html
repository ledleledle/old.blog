Contents

Pertama, kita cek terlebih dahulu apakah Oracle Server sudah berjalan dengan perintah.

@root> /etc/rc.d/oracle-xe status

Jika tidak, saat login kita akan mendapati error seperti contoh berikut

Enter user-name: hr
Enter password:
ERROR:
ORA-01034: ORACLE not available
ORA-27101: shared memory realm does not exist
Linux-x86_64 Error: 2: No such file or directory
Process ID: 0
Session ID: 0 Serial number: 0

Jika Oracle Server belum berjalan atau dapat error seperti itu saat login, jalankan perintah berikut untuk menjalankan servernya.

@root> /etc/rc.d/oracle-xe start

Lalu kita bisa langsung login ke Oracle Sqlplus.

$ sqlplus /nolog
Output 1

Kedua, kita akan mengaktifkan user hr dengan perintah. 

SQL > alter user hr identified by oracle account unlock;

Output 2

Ketiga, login dengan user hr.

Output 3

Permasalahan

User ini (hr) sudah mengandung banyak tabel, sedangkan saya menginginkan database yang fresh, dikarenakan Oracle tidak dapat membuat database baru.

Untuk itu mari kita logout dulu dan membuat user baru.

SQL > exit

Untuk membuat user baru, kita harus masuk menggunakan akun system (kalau MySQL user root).

SQL> connect sys as sysdba

Buat user baru dengan nama dan password sesuai selera (disini saya menggunakan user dan password uts). Dan juga berikan semua akses ke database pada user ini. Berikut perintahnya.

SQL> CREATE USER uts IDENTIFIED BY uts;
SQL> ALTER USER uts QUOTA unlimited ON SYSTEM;
SQL> GRANT CREATE SESSION, CONNECT, RESOURCE, DBA TO uts;
SQL> GRANT ALL PRIVILEGES TO uts;

Sebagai hasilnya, saat kita login dengan user baru. Kita akan mendapatkan database baru yang kosong.

SQL> SELECT * FROM tab;

no rows selected

Berikut komparasi tipe data Oracle dengan MySQL :

  • Komparasi Character antara MySQL dan Oracle
  • Komparasi Numeric antara MySQL dan Oracle
  • Komparasi Datetime antara MySQL dan Oracle

Jadi, kesimpulan saya, Oracle menggunakan format ukuran per bytes dan MySQL menggunakan format ukuran per characters, tapi kalau dipikir-pikir 1 karakter = 1 bytes jadi gaada bedanya. Setelah diketahui kebenaran berikut, kita akan mengambil yang sekiranya sama atau mendekati, tapi jangan lupakan tentang kolom Explanation, kolom tersebut yang akan kita jadikan patokannya. Beberapa tidak ikut ter-screenshot karena terlalu banyak, yang penting saya mengambil yang dibutuhkan saja untuk sekarang. Selengkapnya bisa kalian baca sendiri di Oracle Datatypes dan MySQL Datatypes.

Sooooooooooooooo…… Kita akan list tipe data yang dibutuhkan lalu merubahnya menjadi tipe data Oracle. Berikut daftarnya :

  • INT = akan digantikan dengan NUMERIC.
  • DATE = tetap DATE, karena formatnya sama.
  • VARCHAR = akan digantikan dengan CHAR.
  • TEXT = untuk ini saya kurang yakin, namun kita pilih LONG saja, karena pilihan yang mendekati tipe data TEXT (yang jumlah maksimal karakternya adalah 65.535). LONG memliki ukuran maksimal 2GB (menurut saya kebesaran sih, tapi untuk praktek gakpapa :v ).
Jadi sudah kita putuskan tipe data apa yang akan kita gunakan, sekarang mari kita kerjakan.
